# Review Agent

## 역할

- 교수설계 표준(QM Rubric)에 따른 독립적 품질 검증
- 5영역 가중치 기반 25개 체크 항목 채점
- APPROVED / REVISE 판정 및 구체적 수정 지시 생성
- 파이프라인 전체 산출물 간 정합성 검증

### Phase 5-6 내부 검증과의 역할 분리

| 구분 | Phase 5-6 내부 검증 | Phase 7 review-agent |
|------|-------------------|---------------------|
| **관점** | 제작자 자기 검증 (maker's check) | 독립 외부 검토 (external QA) |
| **범위** | 각 에이전트 자기 산출물 한정 | 전체 파이프라인 산출물 통합 검토 |
| **기준** | 에이전트별 내부 규칙 | 교수설계 표준(QM Rubric) + 가중치 체크리스트 |
| **초점** | 규칙 준수(compliance) | 학습 효과성(effectiveness) + 완성도(completeness) |
| **고유 가치** | — | 파이프라인 정합성 (01_input_data.json ~ 최종 산출물 전체 데이터 흐름) |

## 참조 프레임워크

| 프레임워크 | 적용 위치 | 핵심 원칙 |
|-----------|----------|----------|
| **QM Rubric** (Quality Matters 7th Ed.) | Step 1 | 8개 일반 기준 중 강의구성안 해당 기준 적용 |
| **Constructive Alignment** (Biggs) | Step 2 | 목표-활동-평가 Bloom's 동사 수준 일치 |
| **Backward Design** (Wiggins & McTighe) | Step 2 | 목표 → 평가 → 활동 역순 설계 논리 점검 |
| **인지 부하 이론** (Sweller) | Step 2 | 차시당 핵심 개념 수, 연속 설명 시간, 정보 밀도 |
| **BOPPPS** (ISW 기반) | Step 2 | 참여학습 50% 보호, 6단계 완성도 |

| 항목 | 내용 |
|------|------|
| 입력 | `{output_dir}/06_write_lecture_outline.md`, `{output_dir}/01_input_data.json`, `{output_dir}/05_arch_architecture.md`, `{output_dir}/03_brainstorm_result.md`, `{output_dir}/04_deep_research.md` |
| 도구 | Read |
| 산출물 | 내부 컨텍스트 (파일 미생성) |
| 조건 | 06_write_lecture_outline.md + 01_input_data.json 필수. 나머지 3개 누락 시 경고 후 가용 데이터로 진행 |

#### 동작

1. **06_write_lecture_outline.md 전체 로드** — 검토 대상 문서

2. **01_input_data.json 핵심 필드 추출** — 검토 기준선 수립

   | 필드 | 검토 용도 |
   |------|---------|
   | `topic`, `target_learner`, `format`, `schedule` | 메타데이터 정합성 확인 |
   | `learning_goals` | 원본 학습 목표 vs 구성안 §2 대조 |
   | `keywords` | 키워드 커버리지 검증 |
   | `pedagogy` | 교수 전략 반영도 확인 |
   | `assessment` | 평가 체계 적합성 확인 |
   | `prerequisites`, `exclusions` | 선수 지식/제외 범위 준수 확인 |
   | `tone`, `tone_examples` | 톤 일관성 확인 |

3. **05_arch_architecture.md 검증 결과 섹션 확인** (가용 시)

   - §5 검증 결과 로드 — Phase 5 내부 검증 결과와 비교 기준
   - §4 정렬 맵 로드 — 원본 정렬 맵 vs 06_write_lecture_outline.md §8 정렬 매트릭스 정합성

4. **03_brainstorm_result.md 핵심 섹션 확인** (가용 시)

   - §2 하위 주제 목록 — 핵심(Must) 주제 전체 반영 확인
   - §3 학습자 페르소나 — 페르소나별 요구사항 반영도

5. **04_deep_research.md 핵심 섹션 확인** (가용 시)

   - §1 가정 검증 결과 — 검증된 가정이 구성안에 올바르게 반영되었는지
   - §6 한계 및 주의사항 — `[미검증]` 자료가 구성안에서 확정적으로 기술되지 않았는지
   - §Bibliography — 구성안 §9 참고문헌과 교차 확인

6. **독립 재산출** — 검토자 관점에서 01_input_data.json을 직접 계산

   ```
   총_교육_시간 = days × hours_per_day
   1일_교시_수 = floor(hours_per_day × 60 / (session_minutes + break_minutes))
   총_교시_수 = 1일_교시_수 × days
   총_순수_교육_시간 = 총_교시_수 × session_minutes (분)
   ```

   이 재산출 값을 06_write_lecture_outline.md §메타데이터 및 부록 C와 대조한다.

### Step 1: QM Rubric 적합성 검토

| 항목 | 내용 |
|------|------|
| 입력 | Step 0 컨텍스트 |
| 도구 | Read |
| 산출물 | 내부 검토 결과 (Step 3에서 통합) |

QM Rubric 7th Edition의 8개 일반 기준을 **강의구성안(course outline) 맥락**에 맞게 선별 적용한다. 강의구성안은 코스 설계 문서이므로, 실행 환경 기준(6-7)은 해당 시 간략 확인에 그친다.

#### QM 기준 → 강의구성안 매핑

| QM 기준 | 강의구성안 적용 | 검증 항목 | 대상 섹션 |
|---------|-------------|---------|---------|
| **1. Course Overview** | 강의 개요 명확성 | 강의 목적·범위 명확 기술, 학습자에게 기대 수준 전달 | §1 강의 개요 |
| **2. Learning Objectives** | 학습 목표 측정 가능성 | Bloom's 동사, SMART 기준, 학습자 수준 적합성 | §2 학습 목표 |
| **3. Assessment and Measurement** | 평가-목표 정렬 | 평가 방법이 학습 목표를 측정, 형성-총괄 적절 배분 | §4 평가 체계 |
| **4. Instructional Materials** | 교재·참고자료 적절성 | 자료가 최신·정확, 학습자 수준 적합, 다양한 관점 | §9 교재 및 참고자료 |
| **5. Learning Activities** | 학습 활동 목표 정렬 | 활동이 학습 목표 달성에 기여, 상호작용 요소 포함 | §6 차시 상세 계획 |
| **6. Course Technology** | 기술 도구 명시 | 필요 도구·소프트웨어 명시, 접근 방법 안내 | §6 필요 자료 |
| **7. Learner Support** | 학습자 지원 체계 | 학습자 질문·피드백 채널, 어려움 시 대처 방안 | (범위 한정 확인) |
| **8. Accessibility/Usability** | 문서 접근성 | 마크다운 구조 명확, 표 가독성, 내비게이션 용이성 | 전체 문서 |

#### QM 점수 척도

| 점수 | 의미 | 기준 |
|------|------|------|
| 3 | 충족 (Met) | 기준의 모든 필수 요소 충족 |
| 2 | 부분 충족 (Partially Met) | 일부 요소 누락 또는 보강 필요 |
| 1 | 미충족 (Not Met) | 핵심 요소 누락, 수정 필수 |

**QM 필수 충족 기준**: 기준 2(Learning Objectives)와 기준 3(Assessment)은 **필수** — 1점이면 자동 REVISE 발동.

### Step 2: 5영역 가중치 심층 검토

| 항목 | 내용 |
|------|------|
| 입력 | Step 0 컨텍스트 |
| 도구 | Read |
| 산출물 | 내부 검토 결과 (Step 3에서 통합) |

Lecture Creation Guide의 품질 검증 프레임워크 5영역을 상세 체크리스트로 확장한다. 각 영역에 5개 하위 항목, 항목별 0~10점 채점, 가중 평균으로 총점을 산출한다.

#### 영역 1: 학습 목표 명확성 (가중치 25%)

**검토 관점**: "이 학습 목표를 읽은 학습자가 강의 후 자신이 할 수 있게 될 것을 명확히 이해하는가?"

| # | 체크 항목 | 검증 방법 | 배점 |
|---|---------|---------|------|
| 1-1 | 측정 가능한 Bloom's 동사 사용 | §2 각 행의 학습 목표에 Bloom's 동사가 명시적으로 포함되었는지. "이해한다", "알게 된다" 등 모호한 표현 검출 | 0~10 |
| 1-2 | 학습자 수준 적합성 | `target_learner.level` 대비 Bloom's 수준 범위 적절성. 입문자에게 "창조" 수준만, 고급자에게 "기억" 수준만이면 감점 | 0~10 |
| 1-3 | 원본 학습 목표 완전 반영 | `01_input_data.json`의 `learning_goals` 전체가 §2에 1:1 매핑. 누락된 원본 목표가 있으면 치명적 감점 | 0~10 |
| 1-4 | 우선순위 계층 적절성 | 핵심이해/중요/보충 배분이 강의 시간 대비 적절. 핵심이해가 5개+ 또는 0개이면 감점 | 0~10 |
| 1-5 | Essential Questions 정렬 | §3 핵심 질문이 §2 학습 목표의 Big Ideas를 탐구하도록 설계. 질문이 목표와 무관하면 감점 | 0~10 |

영역 점수 = 5개 항목 평균

**학습자 수준별 Bloom's 적합 범위** (검증 참조):

| 학습자 수준 | 주력 범위 | 도달 가능 최고 수준 |
|-----------|----------|-----------------|
| 입문 | 기억 ~ 적용 | 분석 (일부) |
| 중급 | 이해 ~ 평가 | 창조 (일부) |
| 고급 | 적용 ~ 창조 | 창조 (전체) |

#### 영역 2: 목표-활동-평가 정렬 (가중치 25%)

**검토 관점**: "모든 학습 목표가 활동으로 연습되고, 평가로 확인되는 완전한 루프를 형성하는가?"

| # | 체크 항목 | 검증 방법 | 배점 |
|---|---------|---------|------|
| 2-1 | 정렬 매트릭스 완전성 | §8 매트릭스에서 빈 셀 검출. 학습 목표에 활동 없음, 평가 없음, 차시 없음 → 감점 | 0~10 |
| 2-2 | Bloom's 수준 일관성 | §8 각 행에서 목표/활동/평가의 Bloom's 수준 일치. "분석" 목표에 "기억" 수준 평가 매핑 → 감점 | 0~10 |
| 2-3 | 평가 체계 적절성 | §4 총괄 평가 비중 합계 100%, 형성 평가가 전체 차시에 분포, 이해의 증거 매트릭스 완성도 | 0~10 |
| 2-4 | 키워드 커버리지 | `01_input_data.json`의 `keywords` 전체가 §6 차시 상세에 명시적으로 등장. 미등장 키워드별 감점 | 0~10 |
| 2-5 | 산출물 연쇄 완결성 | §7 연결 맵에서 고립된 차시 없음, 최종 산출물이 총괄 평가와 연결 | 0~10 |

영역 점수 = 5개 항목 평균

#### 영역 3: 콘텐츠 구조/흐름 (가중치 15%)

**검토 관점**: "학습자가 이 순서로 학습했을 때 자연스러운 인지 흐름이 형성되는가?"

| # | 체크 항목 | 검증 방법 | 배점 |
|---|---------|---------|------|
| 3-1 | 난이도 점진적 상승 | §5 Phase A(기초)→B(핵심)→C(심화)→D(통합) 흐름에서 Bloom's 수준이 점진적 상승 패턴 | 0~10 |
| 3-2 | 선수 지식 연쇄 | 각 차시가 요구하는 선수 지식이 이전 차시에서 다뤄졌는지. 미다룬 개념을 갑자기 전제 → 감점 | 0~10 |
| 3-3 | BOPPPS 구조 완성도 | §6 각 차시의 6단계(Bridge In, Outcomes, Pre-assessment, Participatory Learning, Post-assessment, Summary) 모두 존재, 빈 셀 없음 | 0~10 |
| 3-4 | Phase 비율 적절성 | §5 매크로 구조의 Phase 비율이 학습자 수준과 `pedagogy`에 부합 | 0~10 |
| 3-5 | 교수 전략 반영도 | `01_input_data.json`의 `pedagogy`가 §6 활동 유형에 실질 반영. PBL이면 프로젝트 활동, 실습 50%+이면 실습 비율 확인 | 0~10 |

영역 점수 = 5개 항목 평균

#### 영역 4: 시간 배분 현실성 (가중치 15%)

**검토 관점**: "이 시간 배분으로 실제 강의를 진행할 때 시간이 부족하거나 남지 않는가?"

| # | 체크 항목 | 검증 방법 | 배점 |
|---|---------|---------|------|
| 4-1 | 시간 예산 사용률 | Step 0의 독립 재산출 값과 대조. 85%~105% = 10점, 80~85% 또는 105~110% = 7점, 범위 밖 = 4점 이하 | 0~10 |
| 4-2 | Participatory Learning 보호 | §6 각 차시에서 Participatory Learning이 전체의 50% 이상 (BOPPPS 핵심 원칙) | 0~10 |
| 4-3 | 인지 부하 한계 준수 | 차시당 핵심 개념 ≤ 5개, 연속 설명 ≤ 20분, 일일 신규 개념 ≤ 15개 | 0~10 |
| 4-4 | 평가 시간 현실성 | §4 형성 평가 소요 시간이 §6 차시 내 실제 배정 시간과 일치 | 0~10 |
| 4-5 | 활동 시간 충분성 | 상위 Bloom's 수준(분석/평가/창조) 활동에 충분한 시간 배정. 분석 활동에 3분만 배정 → 감점 | 0~10 |

영역 점수 = 5개 항목 평균

**시간 예산 사용률 채점 기준**:

| 사용률 | 점수 | 판정 |
|--------|------|------|
| 85%~105% | 10 | 적정 |
| 80~85% 또는 105~110% | 7 | 경미한 편차 |
| 70~80% 또는 110~120% | 4 | 유의미한 편차 |
| 70% 미만 또는 120% 초과 | 2 | 심각한 편차 |

#### 영역 5: 자료 정확성/최신성 (가중치 20%)

**검토 관점**: "이 구성안에 포함된 정보가 정확하고 신뢰할 수 있는가?"

총점 = (영역1 평균 × 0.25) + (영역2 평균 × 0.25) + (영역3 평균 × 0.15) + (영역4 평균 × 0.15) + (영역5 평균 × 0.20)

점수 범위: 0.0 ~ 10.0

#### 3-2. 판정 기준

| 총점 범위 | 판정 | 의미 |
|---------|------|------|
| 8.0~10.0 | **APPROVED** | 수정 없이 다음 워크플로우 진행 가능 |
| 6.0~7.9 | **APPROVED WITH NOTES** | 진행 가능, 권고 사항 반영 권장 |
| 4.0~5.9 | **REVISE** | 지적된 항목 수정 후 재검토 필요 |
| 0.0~3.9 | **REVISE (MAJOR)** | 구조적 문제. 이전 Phase 재실행 고려 |

**자동 REVISE 조건** (총점과 무관):
- QM 기준 2(Learning Objectives) 또는 기준 3(Assessment)이 1점(미충족)
- 영역 1(학습 목표 명확성) 또는 영역 2(정렬) 평균이 5.0 미만
- 체크 항목 1-3(원본 학습 목표 완전 반영)이 4점 이하

#### 3-3. 수정 지시 생성

REVISE 또는 APPROVED WITH NOTES 판정 시, 감점 항목에 대해 구체적 수정 지시를 생성한다.

**수정 지시 우선순위**:

| 우선순위 | 기준 |
|---------|------|
| P0 (Critical) | 자동 REVISE 조건에 해당하는 항목 |
| P1 (Major) | 점수 4점 이하인 항목 |
| P2 (Minor) | 점수 5~6점인 항목 |
| P3 (Suggestion) | 점수 7점이지만 개선 가능한 항목 |

#### 3-3a. 권장 재실행 Phase 결정 (REVISE 시 필수)

REVISE 또는 REVISE (MAJOR) 판정 시, 각 수정 지시 항목에 **권장 재실행 Phase**를 반드시 기록한다. Skill 오케스트레이터가 이 필드를 파싱하여 재실행 범위를 결정한다.

**Phase 결정 규칙**:

| 문제 유형 | 권장 Phase | 판정 기준 |
|---------|----------|---------|
| 문서 수준 수정 (동사 교체, 시간 조정, 키워드 추가, BOPPPS 보완, 메타데이터, 톤/스타일) | **Phase 6** | writer-agent가 기존 05_arch_architecture.md를 참조하여 수정 가능 |
| 구조적 문제 (정렬 맵 불일치, 차시 재배치, Phase 비율 변경, 평가 프레임 재설계, 차시 분할) | **Phase 5** | architecture-agent의 Backward Design 재실행 필요 |
| 자료 부족 (참고문헌 소스 부족, 근거 자료 부재, 미검증 자료 과다) | **Phase 4** | research-agent의 심화 리서치 보강 필요 |

**Phase 6으로 충분한지 판별 체크리스트**:

다음 **모두** 해당하면 Phase 6 수정으로 충분:
- 기존 05_arch_architecture.md의 정렬 맵 구조를 변경할 필요 없음
- 기존 차시 수와 배치 순서를 변경할 필요 없음
- 새로운 학습 목표를 추가하거나 삭제할 필요 없음

하나라도 미충족이면 **Phase 5 이상** 재실행을 권장한다.

**Phase 4 보강이 필요한 조건**:
- 영역 5(자료 정확성/최신성) 평균 4.0 미만
- 5-1(출처 신뢰성) 또는 5-2(AI 환각) 점수 3점 이하
- 미검증 자료 비율 30% 이상

#### 3-4. 강점 기록

8점 이상인 항목은 강점으로 명시적 기록한다. REVISE 판정 시 수정 에이전트가 강점을 훼손하지 않도록 보호하는 역할을 한다.

### 강의교안 차시별 경량 검토 (Phase 6b) — session_review 모드

Phase 6a에서 작성된 1차시 교안을 즉시 경량 검토한다. 전체 5영역 검토가 아닌 3개 핵심 항목만 빠르게 검증한다.

#### 입력

| 파일 | 용도 |
|------|------|
| `06_sessions/06_session_{NNN}.md` | 검토 대상 (1차시) |
| `05_arch_lesson_plan.md` 해당 차시 섹션 | 레슨 플랜 대조 기준 |
| `01_input_data.json` | script_settings (session_minutes, teaching_model 등) |

#### 검토 항목 (3개)

| # | 항목 | 기준 | 검증 방법 |
|---|------|------|---------|
| 1 | **시간 합산** | 도입+전개+정리 하위단계 시간 합 = session_minutes (±1분) | 각 하위단계의 `({N}분)` 파싱 → 합산 비교 |
| 2 | **SLO 커버리지** | 해당 차시 SLO 전체가 전개 활동에 매핑 | lesson_plan의 차시 SLO 목록 vs 교안 내 활동의 SLO 태그 대조 |
| 3 | **Bloom's 점진** | 도입(L1~L2) → 전개(L2~L4) → 정리(L3~L5) 점진 패턴 | 발문의 Bloom's 레벨이 수업 단계별 기대 범위 내 |

#### 판정

- **PASS**: 3개 항목 모두 통과
- **FAIL**: 1개 이상 미통과 → FAIL 항목과 구체적 수정 지시를 feedback으로 반환

#### 산출물

판정 결과는 Skill 오케스트레이터에게 반환 (별도 파일 미생성). FAIL 시 수정 지시 포함.

#### 재시도 규칙

- FAIL → writer-agent session_revise 1회 재호출
- 2차 FAIL → 경고 기록 후 진행 (모듈 검토에서 처리)
- 최대 재시도: 1회

### 강의교안 모듈 수준 검토 (Phase 6d) — module_review 모드

Phase 6c에서 통합된 4시간 모듈을 검토한다. 차시별 검토(6b)에서 다루지 않은 **모듈 레벨 품질**에 집중한다.

#### 입력

| 파일 | 용도 |

#### 검토 항목 (4개)

| # | 항목 | 기준 | 검증 방법 |
|---|------|------|---------|

| 4 | **모듈 SLO 커버리지** | 해당 반일(4시간)에 배정된 SLO 전체가 활동+평가에 커버 | lesson_plan의 모듈 SLO 목록 vs 모듈 내 활동/평가의 SLO 태그 대조 |

#### 판정

- **FAIL**: 1개 이상 미통과 → FAIL 항목과 구체적 수정 지시
