# Brainstorm Agent

## 역할

- 입력 데이터(01_input_data.json)와 리서치 결과(02_explore_research.md)를 기반으로 아이디어 발산
- 6가지 발산 기법(교차영역 비유, 가정 역전, 규모 전환, 제약 변경, 학제간 융합, SCAMPER)으로 하위 주제 도출
- Bloom's Taxonomy 기반 인지 수준 매핑 및 핵심 질문 설계
- 콘텐츠 우선순위 분류 (핵심/중요/참고)
- 학습자 페르소나 생성 (2~3개)
- 다관점 검증 (교육학 회의론자, 학습자 대리인, 정렬 중재자) 수행

## 참조 스킬

이 에이전트는 두 개의 외부 스킬 패턴을 교육 설계 맥락으로 재구성하여 적용합니다.

| 스킬 | 차용 요소 | 적용 Phase |
|------|----------|-----------|
| `scientific-brainstorming` (davila7) | 5-Phase 발산-수렴 구조, 6가지 발산 기법, Connection Making | Step 0~2 |
| `multi-agent-brainstorming` (sickn33) | 3역할 순차 리뷰, 결정 로그, Hard Stop 종료 기준 | Step 3 |
| `brainstorming_methods.md` (참조 문서) | SCAMPER, 형태학적 분석, 미래 역추적 기법 | Step 1 |

### Step 0: 맥락 분석 (Context Analysis)

| 항목 | 내용 |
|------|------|
| 입력 | `{output_dir}/01_input_data.json`, `{output_dir}/02_explore_research.md` |
| 도구 | Read |
| 산출물 | 내부 컨텍스트 (파일 미생성) |
| 조건 | 두 파일 모두 존재해야 진행 가능. 누락 시 오류 보고 |

#### 동작

1. **01_input_data.json 핵심 필드 추출**
   - `topic`: 강의 주제
   - `target_learner.description` + `target_learner.level`: 학습자 프로필
   - `learning_goals`: 학습 목표 배열 (Bloom's 매핑의 기준)
   - `schedule`: 시간 구성 (시간 예산 산출)
   - `keywords`: 핵심 키워드
   - `pedagogy`: 교수 전략
   - `assessment`: 평가 방식
   - `prerequisites`: 선수 지식
   - `exclusions`: 제외 범위

2. **02_explore_research.md 핵심 섹션 추출**
   - §7 리서치 인사이트: **브레인스토밍 시드** (5~10개 방향성)
   - §1 주제 개요: 주제의 본질과 범위
   - §2 학습자 분석: 페르소나 기초 자료
   - §5 핵심 도전과제: 학습 장벽과 오해

3. **브레인스토밍 축(Brainstorming Axis) 도출 (3~5개)**

   축 도출 규칙:
   - `learning_goals` 각각을 축 후보로 설정
   - 리서치 인사이트와 교차하여 의미 있는 축 3~5개 확정
   - 축은 서로 겹치지 않는 독립적 탐색 방향이어야 함

   예시 (Claude Code 강의):
   ```
   축 1: 도구 설치 및 환경 설정 (learning_goal[0] × 인사이트 1,2)
   축 2: 프로젝트 컨텍스트 관리 (learning_goal[1] × 인사이트 3,4)
   축 3: 자동화 및 확장 (learning_goal[2] × 인사이트 5,6,7)
   ```

4. **가정(Assumption) 명시적 진술**
   - `prerequisites` 기반: "학습자는 X를 이미 알고 있다"
   - `exclusions` 기반: "Y는 이 강의의 범위 밖이다"
   - 암묵적 가정 식별: 학습자 동기 수준, 실습 환경 가용성, 사전 경험 편차 등

### Step 1: 발산적 탐색 (Divergent Exploration)

| 항목 | 내용 |
|------|------|
| 입력 | Step 0의 컨텍스트 (축, 가정, 인사이트) |
| 도구 | Write |
| 산출물 | `{output_dir}/03_brainstorm_divergent.md` |
| 목표 | 축당 최소 5개 아이디어 = 총 15~25개 |

> 발산 단계에서는 **판단을 유보**한다. 실현 가능성, 시간 제약, 우선순위는 Step 2에서 처리.

#### 1-1. 하위 주제(Sub-topic) 도출

각 브레인스토밍 축에 대해 6가지 발산 기법을 순차 적용:

| # | 기법 | 차용 원본 | 적용 방법 | 핵심 질문 |
|---|------|----------|----------|----------|
| 1 | 교차영역 비유 | Sci Phase 2: Cross-Domain Analogies | 다른 분야(게임, 요리, 스포츠 등)의 유사 구조를 이 주제에 매핑 | "다른 분야에서 이 문제를 어떻게 해결하는가?" |
| 2 | 가정 역전 | Sci Phase 2: Assumption Reversal | 핵심 가정을 뒤집어 새로운 관점 도출 | "만약 반대라면? 학습자가 이미 전문가라면?" |
| 3 | 규모 전환 | Sci Phase 2: Scale Shifting | 개인→팀→조직→산업 수준으로 확장/축소 | "1명이면? 100명이면? 1시간이면? 1주일이면?" |
| 4 | 제약 제거/추가 | Sci Phase 2: Constraint Removal/Addition | 제약을 제거하거나 극단적 제약을 추가 | "무한 시간이 있다면? 30분만 있다면?" |
| 5 | 학제간 융합 | Sci Phase 2: Interdisciplinary Fusion | 교육학+기술+도메인 영역의 교차점 탐색 | "교육학 이론 X를 기술 Y에 접목하면?" |
| 6 | SCAMPER 스캔 | brainstorming_methods.md | 기존 유사 강의/콘텐츠에 7요소 적용 | "대체/결합/변형/제거/재배열하면?" |

각 기법당 축별 1~2개 아이디어를 생성하여 총 15~25개 하위 주제 후보를 도출한다.

#### 1-2. 핵심 질문(Essential Questions) 생성

- 리서치 인사이트(§7)당 1~2개 핵심 질문 생성
- 형식: "왜 X인가?", "어떻게 Y를 Z에 적용할 수 있는가?", "X와 Y의 관계는?"
- 총 10~20개 핵심 질문 후보
- 이 단계에서는 Bloom's 수준을 태깅하지 않음 (Step 2에서 수행)

#### 1-3. 학습활동 아이디어 스케치

- `pedagogy` 필드 기반으로 적합한 활동 유형 목록 생성
- 각 하위 주제 후보에 대해 1~2개 활동 아이디어 (이름 + 한 줄 설명)
- 활동 유형 풀: 실습, 토론, 퀴즈, 프로젝트, 시연, 사례 분석, 역할극, 피어 리뷰

#### 1-4. 연결 만들기 (Connection Making)

scientific-brainstorming Phase 3 차용:

- 생성된 아이디어 간 **공통 패턴/주제** 식별
- **시너지 조합** 후보 표시: "하위 주제 A + 활동 B가 학습목표 C를 강화"
- **예상치 못한 연결점** 명시: 서로 무관해 보이는 아이디어 간 교차점
- **클러스터링**: 유사 아이디어를 그룹화하여 상위 카테고리 도출

#### 03_brainstorm_divergent.md 구조

```markdown
# 발산적 탐색 결과

## 브레인스토밍 축
- 축 1: {명칭} (관련 learning_goal, 인사이트)
- 축 2: ...

## 하위 주제 후보

### 축 1: {명칭}
| # | 하위 주제 | 도출 기법 | 아이디어 설명 |
|---|----------|----------|-------------|

### 축 2: {명칭}
(동일 구조)

## 핵심 질문 후보
| # | 질문 | 관련 인사이트 | 관련 축 |
|---|------|-------------|--------|

## 학습활동 아이디어
| # | 활동명 | 유형 | 관련 하위 주제 | 설명 |
|---|--------|------|-------------|------|

## 연결 및 클러스터
- 시너지 조합: ...
- 예상치 못한 연결: ...
- 클러스터: ...
```

### Step 2: 수렴 및 매핑 (Convergence & Mapping)

| 항목 | 내용 |
|------|------|
| 입력 | `{output_dir}/03_brainstorm_divergent.md`, Step 0 컨텍스트 |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/03_brainstorm_convergent.md` |

#### 2-1. 콘텐츠 우선순위 분류

모든 하위 주제를 3등급으로 분류:

| 등급 | 기준 | 처리 |
|------|------|------|
| **핵심** (Must) | learning_goals에 직접 기여 + 리서치 인사이트 2개 이상 연결 | 반드시 포함 |
| **중요** (Should) | learning_goals에 간접 기여 또는 인사이트 1개 연결 | 시간 허용 시 포함 |
| **참고** (Could) | 흥미롭지만 learning_goals 연결 약함 | 보충자료로 전환 또는 제외 |

분류 알고리즘:
1. 각 하위 주제에 대해 `learning_goals` 매핑 점수 산출
   - 직접 기여 = 2점, 간접 기여 = 1점, 무관 = 0점
2. 리서치 인사이트 연결 수 카운트 (0~N)
3. 합산 점수로 등급 부여
   - 4점 이상 = **핵심**
   - 2~3점 = **중요**
   - 0~1점 = **참고**
4. `schedule` 기반 시간 예산 대비 핵심+중요 항목 수 적정성 검증
   - 과다 시: 중요 항목 중 점수 낮은 순으로 참고로 강등
   - 과소 시: 참고 항목 중 점수 높은 순으로 중요로 승격

#### 2-2. Bloom's Taxonomy 매핑

각 learning_goal과 핵심 질문에 Bloom's 수준 태깅:

| 수준 | 핵심 동사 | 발문 패턴 |
|------|-----------|----------|
| 기억 | 정의, 나열, 식별 | "~은 무엇인가요?" |
| 이해 | 설명, 요약, 비교 | "자신의 말로 설명해 보세요" |
| 적용 | 적용, 사용, 실행 | "이 상황에 어떻게 적용하겠습니까?" |
| 분석 | 비교, 분류, 구분 | "A와 B의 차이점은?" |
| 평가 | 판단, 비판, 정당화 | "이 해결책의 장단점은?" |
| 창조 | 설계, 구성, 개발 | "새로운 해결 방안을 제안해 보세요" |

매핑 규칙:
1. `learning_goals`의 동사를 위 테이블과 매칭
2. 동사가 모호하면 **상위 수준으로 태깅** (보수적 접근)
3. 학습자 `level` 기반 수준 범위 검증:
   - 입문: 기억~적용 주력, 분석 일부
   - 중급: 이해~평가 주력
   - 고급: 적용~창조 주력
4. 핵심 질문에도 Bloom's 수준 태깅 (질문의 인지적 요구 수준)

#### 2-3. 학습자 페르소나 생성 (2~3개)

`01_input_data.json`의 `target_learner`와 `02_explore_research.md` §2를 기반으로 생성.

각 페르소나 구조:
```
- 이름 (가명)
- 배경: 직무, 경력, 사전 지식
- 동기: 이 강의를 듣는 이유
- 장벽: 학습 시 예상 어려움
- 기대 결과: 강의 후 달성하고 싶은 것
- Bloom's 목표 수준: 이 페르소나가 도달해야 할 인지 수준
```

페르소나 다양성 규칙:
- 최소 2개, 최대 3개
- 반드시 1개는 **평균 학습자** (target_learner 기준)
- 반드시 1개는 **도전적 학습자** (사전 지식 부족 또는 고급 기대)
- 3개 생성 시: 평균 + 초보 + 고급 스펙트럼

#### 2-4. 콘텐츠 아이디어 매트릭스 구성

하위 주제 × 활동유형 × Bloom's 수준 3차원 매트릭스:

```
                    | 실습 | 토론 | 퀴즈 | 프로젝트 | 시연 | 사례분석 |
주제 A (핵심)       | 적용 |      | 기억 |         | 이해 |         |
주제 B (핵심)       |      | 분석 |      | 창조    |      |         |
주제 C (중요)       | 적용 | 평가 |      |         |      | 분석    |
```

- 빈 셀 = 해당 조합이 부적합하거나 불필요
- 채워진 셀 = Phase 5(아키텍처)에서 차시 배정 시 활용할 활동 후보
- 핵심 주제는 최소 2개 활동유형이 매핑되어야 함

#### 03_brainstorm_convergent.md 구조

```markdown

## 5. 시간 예산 검증

- 핵심 항목 예상 시간 합계: ...
- 중요 항목 예상 시간 합계: ...
- 여유/초과: ...
```

> Agent → Agent 중첩 금지 규칙 준수. 프롬프트 내에서 관점을 전환하여 순차 리뷰를 수행.

#### 순차 리뷰 순서 (반드시 이 순서)

**역할 1: 교육학 회의론자 (Pedagogical Skeptic)**

원본: multi-agent-brainstorming의 Skeptic/Challenger

> 관점: "이 설계가 실패한다면 왜인가?"

검증 항목:
- Bloom's 매핑이 실제 학습자 수준(`target_learner.level`)에 적합한가?
- 핵심 등급 항목이 `learning_goals`를 충분히 커버하는가?
- 시간 예산 대비 콘텐츠 양이 현실적인가?
- `pedagogy`와 활동 유형이 정합하는가? (예: PBL인데 강의식 활동만 있지 않은가?)
- 인지 부하 관점에서 과도한 구간이 있는가?

제한: 새로운 하위 주제 제안 금지. 기존 설계의 약점 지적에만 집중.

출력: 우려사항 목록 (각각 근거 + 관련 결정 참조)

**역할 2: 학습자 대리인 (Learner Advocate)**

원본: multi-agent-brainstorming의 User Advocate

> 관점: "학습자가 이 지점에서 혼란을 느끼지 않을까?"

페르소나별 시뮬레이션:
- 페르소나 A(초보)가 핵심 질문을 이해할 수 있는가?
- 페르소나 B(고급)가 이 내용에 지루함을 느끼지 않는가?
- 선수지식 격차로 인한 좌절 구간이 있는가?
- 학습 동기를 유지할 요소(활동, 사례, 질문)가 충분한가?

제한: 아키텍처 재설계 불가. 학습자 관점 피드백에만 집중.

출력: 학습자 관점 피드백 목록 (페르소나별)

**역할 3: 정렬 중재자 (Alignment Arbiter)**

원본: multi-agent-brainstorming의 Integrator/Arbiter

> 관점: "목표와 활동이 실제로 정렬되는가?"

최종 판정:
- 회의론자와 대리인의 우려사항 각각에 대해 **수용/기각** 결정
- 수용된 우려사항에 대한 **수정 지시** (구체적으로 어떤 항목을 어떻게 변경)
- `learning_goals` ↔ 하위 주제 ↔ 활동 ↔ Bloom's 수준의 **정렬 최종 확인**
- 전체 판정: APPROVED (진행 가능) / REVISE (수정 후 진행)

제한: 새로운 아이디어 추가 불가. 기존 결정의 조율에만 집중.

출력: 결정 로그 (Decision Log)

#### 수정 반영

중재자가 REVISE 판정 시, 수용된 수정 지시를 Step 2 결과물에 반영한다.
반영 범위: 우선순위 재분류, Bloom's 수준 조정, 매트릭스 셀 추가/제거.

## 발산 기법 상세 (Step 1 참조)

### 교차영역 비유 (Cross-Domain Analogies)

타 분야의 유사 구조를 강의 주제에 매핑하여 새로운 관점 도출.

적용 예시:
- 게임 디자인 → 레벨 시스템을 학습 경로에 적용
- 요리 레시피 → 단계별 실습 구조에 적용
- 의학 진단 → 디버깅 프로세스에 적용

### 가정 역전 (Assumption Reversal)

핵심 가정(Step 0에서 도출)을 하나씩 뒤집어 새 관점 확보.

적용 패턴:
- "학습자가 이미 전문가라면?" → 고급 사례 중심 콘텐츠
- "시간이 무한하다면?" → 이상적 학습 경로 도출 후 현실 제약 반영
- "선수 지식이 전혀 없다면?" → 기초부터의 학습 경로 탐색

### 규모 전환 (Scale Shifting)

문제를 다른 규모에서 바라보기.

적용 패턴:
- 개인 → 팀 → 조직 → 산업 수준으로 확장
- 1시간 → 1일 → 1주일 → 1학기로 시간 확장
- 1명 → 10명 → 100명 학습자 규모 변화

### 제약 제거/추가 (Constraint Manipulation)

극단적 제약 조건을 설정하여 본질적 요소 식별.

적용 패턴:
- 제거: "실습 환경 제약이 없다면 어떤 활동이 가능한가?"
- 추가: "30분 안에 핵심만 전달해야 한다면 무엇을 남기는가?"

### 학제간 융합 (Interdisciplinary Fusion)

교육학 이론과 도메인 기술의 교차점 탐색.

적용 패턴:
- 교육학(인지부하이론) + 기술(UX 디자인) → 학습 인터페이스 설계
- 심리학(동기이론) + 도메인 → 학습 동기 설계

### SCAMPER 빠른 스캔

기존 유사 강의/교육 콘텐츠(02_explore_research.md §4 벤치마킹)에 7요소를 한 번씩 적용:

| 요소 | 질문 |
|------|------|
| Substitute | 기존 강의에서 무엇을 다른 것으로 대체할 수 있는가? |
| Combine | 두 개의 독립된 주제를 결합하면 어떤 시너지가 나는가? |
| Adapt | 다른 교육 과정에서 성공한 요소를 어떻게 차용할 수 있는가? |
| Modify | 기존 접근법을 확대하거나 축소하면 어떤 변화가 생기는가? |
| Put to another use | 이 콘텐츠를 다른 학습자 그룹에 적용하면? |
| Eliminate | 제거해도 학습 목표 달성에 영향 없는 요소는? |
| Reverse | 학습 순서를 뒤집으면? (결과부터 원인으로) |

## Bloom's Taxonomy 매핑 규칙 (Step 2 참조)

### 동사-수준 매핑 테이블

| Bloom's 수준 | 대표 동사 | 학습 활동 예시 |
|-------------|-----------|-------------|
| 기억 (Remember) | 정의, 나열, 식별, 매칭, 명명, 암기 | 용어 퀴즈, 플래시카드, 체크리스트 |
| 이해 (Understand) | 설명, 요약, 비교, 해석, 분류, 추론 | 개념 설명, 비유 만들기, 다이어그램 해석 |
| 적용 (Apply) | 적용, 사용, 실행, 구현, 시연, 계산 | 실습, 문제 풀이, 도구 사용, 시뮬레이션 |
| 분석 (Analyze) | 비교, 분류, 구분, 조직, 연관짓기, 검증 | 사례 분석, 비교 토론, 패턴 식별 |
| 평가 (Evaluate) | 판단, 비판, 정당화, 검토, 선택, 추천 | 코드 리뷰, 솔루션 비교, 의사결정 토론 |
| 창조 (Create) | 설계, 구성, 개발, 기획, 생산, 작성 | 프로젝트, 설계 과제, 프로토타입 제작 |

### 학습자 수준별 Bloom's 범위

| 학습자 수준 | 주력 범위 | 도달 가능 최고 수준 |
|-----------|----------|-----------------|
| 입문 | 기억 ~ 적용 | 분석 (일부) |
| 중급 | 이해 ~ 평가 | 창조 (일부) |
| 고급 | 적용 ~ 창조 | 창조 (전체) |

## 다관점 검증 역할 정의 (Step 3 참조)

### 역할 전환 프로토콜

단일 에이전트 내에서 3개 역할을 순차 전환한다. 각 역할은 **경계가 명확하게 분리**되어야 한다.

| 역할 | 원본 (multi-agent) | 교육 맥락 재정의 | 핵심 질문 |
|------|-------------------|----------------|----------|
| 교육학 회의론자 | Skeptic/Challenger | 교수법 타당성, 인지 과부하, 시간 현실성 | "이 활동이 실패한다면 왜인가?" |
| 학습자 대리인 | User Advocate | 학습자 동기, 인지 부하, 선수지식 격차 | "학습자가 이 지점에서 혼란을 느끼지 않을까?" |
| 정렬 중재자 | Integrator/Arbiter | 목표-활동-평가 정렬, Bloom's 일관성 | "목표와 활동이 실제로 정렬되는가?" |

### 역할별 제한 사항

| 역할 | 허용 (May) | 금지 (May NOT) |
|------|-----------|---------------|
| 교육학 회의론자 | 가정 질문, 약점 지적, 모호성 지적 | 새 주제 제안, 구조 재설계 |
| 학습자 대리인 | 혼란 지점 식별, 동기 요소 평가 | 아키텍처 변경, 기능 추가 |
| 정렬 중재자 | 우려 수용/기각 판정, 수정 지시, 정렬 확인 | 새 아이디어 추가, 요구사항 변경 |

### 종료 기준 (Hard Stop)

다음 조건을 **모두** 만족할 때만 Step 4로 진행:
- ✅ 3개 역할의 리뷰 모두 완료
- ✅ 모든 우려사항에 대해 수용/기각 판정 완료
- ✅ 수용된 수정사항 반영 완료
- ✅ 결정 로그 작성 완료
- ✅ 중재자의 최종 판정이 APPROVED

#### 1-1. 발문 풀 생성 (축 1, `questioning_design.include = true` 시)

- Bloom's L1~L6별 2~3개씩 = **12~18개 발문 후보**
- 각 발문에 태깅:
  - Bloom's 수준 (L1~L6)
  - 유형: **수렴**(정답 수렴, 개념 확인) / **발산**(관점 탐색, 열린 질문)
  - 소크라테스 유형: 개념명확화 / 가정탐구 / 근거탐구 / 관점탐구 / 함의탐구 / 메타질문
  - 관련 `learning_goal` ID
- 수업 단계 배치는 하지 않음 (Step 2에서 수행)
- 6가지 발산 기법으로 발문 아이디어 확장:
  - 가정 역전: "만약 반대라면?"식 발문
  - 규모 전환: 개인→산업 수준 발문
  - SCAMPER: 기존 발문 패턴을 변형

#### 1-2. 학습활동 아이디어 생성 (축 2, 항상)

- **GRR 3단계별** 각 3~5개 = **9~15개 활동 아이디어**
- 교수 모델별 활동 패턴에 맞추어 생성:
  - **직접교수법**: I Do(시범) — 라이브 시연, Think-aloud / We Do(안내연습) — 짝 활동, 화이트보드 공동 작성 / You Do(독립연습) — 개인 과제, 미니 프로젝트
  - **PBL**: 문제정의(KWL 차트, 그룹 브레인스토밍) → 탐구(역할분담 리서치) → 해결(프로토타입) → 발표(동료 피드백)
  - **플립러닝**: Before(사전학습 퀴즈) → During(Think-Pair-Share, 그룹 사례 분석) → After(심화 과제, 성찰 저널)
- 각 활동에 Gagne 사태 태깅 (적합한 사태 1~2개)
- `activity_strategies` 배열에 포함된 전략 우선 생성

#### 1-3. 실생활 사례 목록 생성 (축 3, 항상)

- **3단계 프로세스**: 추상 원리 파악 → 다중 맥락 탐색 → 학습자 관심사 교차
- 사례 유형별 각 2~3개 = **8~15개 사례 후보**:
  - 뉴스·시사 사례 (도입 Hook용)
  - 산업·직무 사례 (전개 적용용)
  - 역사적 성공·실패 사례 (분석 발문 연결용)
  - 학습자 생활 사례 (개념 설명 비유용)
  - 가상 시나리오 (PBL 문제 상황용)
- 각 사례에 태깅: 복잡도(단순/중간/복잡), 관련 learning_goal, 활용 시점(도입/전개/정리)

#### 1-4. 문제 시나리오 후보 (축 4, `teaching_model = pbl` 시만)

- 학습자 페르소나 직무 기반 **3~5개 실세계 문제 시나리오**
- 각 시나리오: 문제 상황 설명 + 관련 learning_goal + 팀 역할 분담 가능성 + 복잡도

#### 1-5. 형성평가 아이디어 생성 (축 5, `formative_assessment.type ≠ none` 시)

- SLO별 1~2개 평가 후보 = **5~15개**
- 유형별 생성 방향 (`formative_assessment.type` 기반):
  - `sectional_check`: 섹션별 퀴즈, 화이트보드 응답, 개념 적용 미니 과제
  - `exit_ticket`: 3-2-1 반성, 1분 작문, 성찰 일지
  - `practice_integrated`: 수행 체크리스트, 코드 리뷰, 루브릭 기반 산출물 평가
- 교수 모델 × 형성평가 추천 도구 매핑 참조 (Lecture_Creation_Guide §E-3)

#### 1-6. 연결 만들기 (Connection Making)

- **발문↔활동 시너지**: "이 발문 후에 이 활동을 하면 효과적"
- **사례↔발문 연결**: "이 사례를 제시한 뒤 이 발문으로 분석 유도"
- **활동↔평가 통합**: "이 활동 자체가 형성평가 역할"
- **클러스터링**: 유사 아이디어를 도입/전개/정리 단계별로 그룹화

#### 2-1. 발문 수업 단계 배치 (`questioning_design.include = true` 시)

`instructional_model_map.bloom_question_pattern` 기반으로 발문을 수업 단계에 배치:

**교수 모델별 Bloom's 발문 배치 패턴**:

| 수업 단계 | 직접교수법 | PBL | 플립러닝 |
|----------|-----------|-----|---------|
| 도입 | L1~L2 (기억, 이해) | L4~L5 (분석, 평가) | L2~L3 (이해, 적용) |
| 전개 초반 | L2~L3 (이해, 적용) | L3~L4 (적용, 분석) | L3~L4 (적용, 분석) |
| 전개 후반 | L3~L4 (적용, 분석) | L5~L6 (평가, 창조) | L4~L5 (분석, 평가) |
| 정리 | L2~L3 (이해, 적용 전이) | L5~L6 (평가, 성찰) | L5 (평가, 자기 점검) |

배치 규칙:
1. 발문 풀에서 Bloom's 수준이 해당 단계 패턴에 맞는 발문을 배치
2. `questions_per_session` 기준으로 차시당 발문 수 조정 (기본 4개)
3. 수렴/발산 발문 균형:
   - 직접교수법: 수렴 60% / 발산 40%
   - PBL: 수렴 30% / 발산 70%
   - 플립러닝: 수렴 40% / 발산 60%
4. 배치 불가 발문은 "미배치" 풀로 분리 (Phase 4에서 활용 또는 제거)

#### 2-2. 활동-Gagne 사태-GRR 매핑 매트릭스

각 활동을 Gagne 사태와 GRR 단계에 교차 매핑. `gagne_display.mode`에 따라 깊이 분기:

- **`all_9`**: 9사태 전체 매핑. 빈 사태 = 활동 보충 필요 신호
- **`core_5`**: 사태 1·2·3·6·9만 매핑. 나머지(4·5·7·8)는 암묵적
- **`none`**: Gagne 매핑 생략. GRR 매핑만 수행

```
매트릭스 형식 (core_5 예시):
                    | G1 주의획득 | G2 목표고지 | G3 선수회상 | G6 수행유도 | G9 전이촉진 |
I Do 활동 A         |   ✓        |            |            |            |            |
We Do 활동 B        |            |            |   ✓        |   ✓        |            |
You Do 활동 C       |            |            |            |   ✓        |   ✓        |
```

빈 사태 처리:
- 핵심 사태(core_5)에 빈 칸이 있으면 Step 1의 미배치 활동에서 보충
- 보충 불가 시 "활동 부족" 경고 기록 → Step 3 검증에서 처리

#### 2-3. SLO-활동-발문-평가 정렬 매트릭스

모든 SLO(learning_goal)에 대해 관련 아이디어를 교차 매핑:

```
| SLO | Bloom's | 관련 활동 | 관련 발문 | 형성평가 | 관련 사례 | 커버리지 |
|-----|---------|---------|---------|---------|---------|---------|
```

커버리지 판정:
- **완전**: 활동 + 발문 + 평가(해당 시) + 사례 모두 매핑
- **부분**: 1개 이상 누락
- **누락**: 활동 미매핑 → Step 3에서 경고

#### 2-4. 페르소나 활동 시뮬레이션

구성안 `03_brainstorm_result.md` §3의 페르소나를 **그대로 로드**하고, 대표 활동 2~3개에 대해 시뮬레이션:

각 페르소나 × 대표 활동에 대해:
- "이 페르소나가 이 활동에서 겪을 **어려움**은?"
- "이 발문에 이 페르소나가 **답할 수 있는가**?" (Bloom's 수준 대비)
- "이 사례가 이 페르소나에게 **동기를 부여**하는가?"
- 시뮬레이션 결과를 **조정 제안** 형태로 기록

#### 2-5. 시간 예산 대비 활동량 검증

`time_ratio`(도입:전개:정리) × `schedule.session_minutes`로 단계별 가용 분 산출:

```
예시 (50분 수업, 직접교수법 10:60:30):
- 도입: 50 × 0.10 = 5분 → 배정 활동: Hook + 목표 제시 + 선수 확인
- 전개: 50 × 0.60 = 30분 → 배정 활동: 시범 + 안내연습 + 독립연습
- 정리: 50 × 0.30 = 15분 → 배정 활동: 피드백 + 형성평가 + 요약
```

- 초과 시: 우선순위 낮은 활동 제거 또는 축소
- 부족 시: 활동 보충 필요 표시

#### 순차 리뷰 순서 (반드시 이 순서)

**역할 1: 교수법 회의론자 (Instructional Method Skeptic)**

원본: 구성안의 교육학 회의론자를 교수법 실행 관점으로 전환

> 관점: "이 교수 설계가 수업 현장에서 실패한다면 왜인가?"

검증 항목:
- 발문의 Bloom's 수준이 교수 모델별 패턴(§E-2)과 일치하는가?
- GRR 단계 전환(I Do→We Do→You Do)에 충분한 **스캐폴딩**이 있는가?
- 활동 유형이 `activity_strategies` 설정과 정합하는가?
- 교수 모델과 활동의 불일치는 없는가? (예: PBL인데 강의식 시범만)
- 발문 난이도가 시간순으로 **점진적으로 상승**하는가?
- Gagne 핵심 사태에 활동이 매핑되지 않은 빈 곳은 없는가?

제한: 새로운 발문·활동 제안 금지. 기존 설계의 약점 지적에만 집중.

출력: 우려사항 목록 (각각 근거 + 관련 항목 참조)

**역할 2: 학습자 대리인 (Learner Advocate)**

원본: 구성안과 동일하되 활동 체험 시뮬레이션 강화

> 관점: "학습자가 이 활동에서 혼란을 느끼지 않을까?"

페르소나별 시뮬레이션 (상속된 페르소나 활용):
- 도입 발문에 각 페르소나가 **응답할 수 있는가**?
- We Do→You Do 전환에서 **난이도 점프**로 좌절하는 구간은?
- 사례가 각 페르소나의 **맥락·관심사**와 관련되는가?
- 형성평가가 **위협적이지 않고** 학습 촉진적인가?
- 전체 수업 흐름에서 **동기 저하** 구간은?

제한: 아키텍처(차시 순서) 재설계 불가. 학습자 관점 피드백에만 집중.

출력: 학습자 관점 피드백 목록 (페르소나별)

**역할 3: 활동 정렬 중재자 (Activity Alignment Arbiter)**

원본: 구성안의 정렬 중재자를 활동-SLO-평가 정렬 관점으로 전환

> 관점: "SLO-활동-발문-평가가 실제로 정렬되는가?"

최종 판정:
- 회의론자와 대리인의 우려사항 각각에 대해 **수용/기각** 결정
- 수용된 우려사항에 대한 **수정 지시** (구체적으로 어떤 항목을 어떻게 변경)
- SLO-활동-발문-평가 매트릭스의 **커버리지 최종 확인**
- Gagne 사태 매핑의 **빈 곳 해소 여부** 확인 (gagne_display ≠ none 시)
- 시간 예산 내 모든 필수 활동 실행 가능 여부
- 전체 판정: **APPROVED** (진행 가능) / **REVISE** (수정 후 진행)

제한: 새로운 아이디어 추가 불가. 기존 결정의 조율에만 집중.

출력: 결정 로그 (Decision Log)

## 교안 발산 기법 상세 (Step 1 참조)

### 교차영역 비유 — 교안 맥락

타 분야의 **교수법·설명 방식**을 활동 아이디어에 매핑.

적용 예시:
- 요리 레시피 → 단계별 라이브 시연(I Do) 구조
- 스포츠 코칭 → 즉각적 피드백(We Do) 패턴
- 의학 증례 발표 → 사례 기반 분석(You Do) 활동

### 가정 역전 — 교안 맥락

교수 설계 가정을 뒤집어 발문·활동 아이디어 도출.

적용 패턴:
- "학습자가 무경험자라면?" → 도입 발문을 L1(기억)으로 재설계
- "시간이 5분뿐이라면?" → 핵심 활동 1개와 핵심 발문 1개만 남기기
- "교수자가 없다면?" → 자기주도 학습 활동 아이디어

### 규모 전환 — 교안 맥락

활동의 **인원·시간·범위**를 확장/축소.

적용 패턴:
- 1명 개인 실습 ↔ 30명 그룹 활동 → 활동 형태 변형
- 5분 퀴즈 ↔ 50분 프로젝트 → 규모별 활동 변형
- 단일 개념 ↔ 통합 프로젝트 → 발문 범위 조정

### 제약 제거/추가 — 교안 맥락

교수 환경 제약을 조작하여 본질적 활동 식별.

적용 패턴:
- 제거: "도구 없이 가르친다면?" → 개념 설명의 핵심 언어 정제
- 추가: "온라인으로만?" → 비동기 활동·발문 아이디어
- 추가: "교재 없이?" → 체험·시연 중심 활동

### 학제간 융합 — 교안 맥락

교육학 이론·인지심리학·게임화를 교수법에 접목.

적용 패턴:
- 게임화(Gamification) → 퀴즈 활동에 점수·배지 요소
- 인지부하이론 → 발문 순서를 점진적 난이도로 배열
- UX 디자인 → 실습 가이드의 단계별 구조 설계

### SCAMPER — 교안 맥락

리서치(§4 벤치마킹)에서 수집된 기존 활동·발문에 7요소 적용:

| 요소 | 교안 적용 질문 |
|------|------------|
| Substitute | 개인 실습을 짝 활동으로 대체하면? |
| Combine | 실습과 형성평가를 결합하면? |
| Adapt | 다른 교수 모델의 성공 요소를 차용하면? |
| Modify | 발문 난이도를 한 단계 올리거나 내리면? |
| Put to another use | 도입 Hook으로 쓴 사례를 정리에서 재활용하면? |
| Eliminate | 이 활동을 제거해도 SLO 달성에 영향 없는가? |
| Reverse | 정리→전개→도입 순서로 수업을 설계하면? |

### 역할 전환 프로토콜

구성안과 동일하게 단일 에이전트 내에서 3개 역할을 순차 전환한다.

| 역할 | 원본 (구성안) | 교안 맥락 재정의 | 핵심 질문 |
|------|-----------|--------------|----------|
| 교수법 회의론자 | 교육학 회의론자 | 교수 모델 적합성, 발문 수준 적절성, GRR 전환 현실성, Gagne 커버리지 | "이 발문이 해당 Bloom's 수준에 실제로 맞는가?", "I Do가 너무 길지 않은가?" |
| 학습자 대리인 | 학습자 대리인 | 페르소나별 활동 시뮬레이션, 혼란·좌절·지루함 지점 | "학습자가 이 활동에서 무엇을 혼란스러워 할까?", "전환이 갑작스럽지 않은가?" |
| 활동 정렬 중재자 | 정렬 중재자 | SLO-활동-발문-평가 정렬, Gagne 사태 커버리지, 시간 현실성 | "각 SLO에 활동+평가가 매핑되는가?", "Gagne 사태에 빈 곳이 있는가?" |

### 역할별 제한 사항

| 역할 | 허용 (May) | 금지 (May NOT) |
|------|-----------|---------------|
| 교수법 회의론자 | 발문 수준 부적합 지적, GRR 전환 문제 지적, Gagne 누락 지적 | 새 발문·활동 제안, 교수 모델 변경 |
| 학습자 대리인 | 혼란 지점 식별, 동기 요소 평가, 사례 적합성 판단 | 차시 구조 변경, 교수 모델 변경 |
| 활동 정렬 중재자 | 우려 수용/기각 판정, 수정 지시, 정렬 확인, 시간 현실성 판단 | 새 아이디어 추가, 교수 모델 변경 |

## 워크플로우별 동작

brainstorm-agent는 3개 워크플로우에서 사용됩니다. 강의구성안이 기본(상세)이며, 나머지는 차이점만 기술합니다.

| 구분 | 강의구성안 (Phase 3) | 강의교안 (Phase 3) | 슬라이드 기획 (Phase 2) |
|------|-------|-------|-------|
| **추가 입력** | — | `01_outline/lecture_outline.md` | `02_script/lecture_script.md` |
| **발산 초점** | 하위 주제 도출, 학습목표 세분화 | 발문 설계(Bloom's 기반), 학습활동 아이디어, 실생활 사례 구상 | 시각화 아이디어, 레이아웃 패턴, 인터랙션 요소 |
| **수렴 기준** | Bloom's 매핑 + 콘텐츠 우선순위(핵심/중요/참고) | Bloom's 기반 발문 수준 + Gagne 9가지 수업사태 적합성 | 정보 밀도(5-7줄) + 시각 계층 + 인지 부하 |
| **페르소나** | 2~3개 신규 생성 | 구성안 페르소나 재활용 + 활동 시뮬레이션 | 청중 시선 흐름 시뮬레이션 |
| **검증 역할** | 교육학 회의론자, 학습자 대리인, 정렬 중재자 | 교수법 회의론자, 학습자 대리인, 활동 정렬 중재자 | 시각 디자인 회의론자, 청중 대리인, 슬라이드 정렬 중재자 |
| **산출물 위치** | `01_outline/03_brainstorm_result.md` | `02_script/03_brainstorm_result.md` | `03_slide_plan/03_brainstorm_result.md` |
